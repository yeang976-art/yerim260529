# Backend Motivation

## Why Backend

백엔드 개발은 사용자의 요청이 서버로 전달되고, 데이터가 처리되어 다시 응답으로 돌아오는 흐름을 설계하는 일이라고 생각합니다.

프론트엔드가 사용자에게 보이는 화면과 상호작용을 담당한다면, 백엔드는 그 요청을 처리하고 필요한 데이터를 다루며 서비스의 중심 흐름을 안정적으로 유지하는 역할을 한다고 이해하고 있습니다.

## Interest Point

API, 데이터베이스, 서버 로직처럼 눈에 직접 보이지 않지만 서비스가 실제로 동작하게 만드는 부분에 관심이 있습니다.

특히 API 명세서 작성과 Postman 실습을 하면서, 요청과 응답이 명확하게 정의되어야 프론트엔드와 백엔드가 같은 기준으로 개발할 수 있다는 점을 알게 되었습니다.

## Current Learning

현재는 Java/Spring 백엔드 부트캠프를 통해 HTTP, API, Git, 데이터베이스, Spring 기반 웹 개발의 기본기를 학습하고 있습니다.

최근에는 Chrome DevTools Network 탭에서 OPTIONS 요청과 GET 요청을 구분하며, API를 분석할 때 URL만 보는 것이 아니라 Method, Header, Status Code, Response를 함께 확인해야 한다는 점을 배웠습니다.

## Growth Direction

앞으로 Java/Spring, JPA, Security, 배포, 팀 프로젝트 경험을 쌓으며 실제 서비스 흐름을 설계할 수 있는 백엔드 개발자로 성장하고 싶습니다.

단순히 기능을 구현하는 것에서 끝나지 않고, 요청이 들어오고 데이터가 처리되며 응답으로 돌아가는 전체 흐름을 이해하고 설명할 수 있는 개발자가 되는 것이 목표입니다.
