# Git Collaboration Flow

## Overview

이 문서는 Git과 GitHub를 활용해 브랜치를 나누고, Pull Request를 통해 변경 사항을 병합하는 협업 흐름을 정리한 기록입니다.

이번 실습의 핵심은 main 브랜치에 직접 작업하지 않고, 주제별 브랜치를 만들어 변경 사항을 관리하는 것이었습니다.

## Branch Strategy

이번 저장소에서는 다음과 같이 브랜치를 나누어 작업했습니다.

| Branch | Role |
| --- | --- |
| `main` | 최종 결과물이 병합되는 기본 브랜치 |
| `profile-foundation` | 기본 자기소개와 현재 학습 상태 정리 |
| `backend-motivation` | 백엔드 개발자를 목표로 하는 이유 정리 |
| `backend-practice-archive` | 부트캠프 실습과 과제 기록 아카이브 |
| `git-collaboration-flow` | Git/GitHub 협업 흐름 정리 |
| `learning-system` | TIL, Notion, 기술 블로그 기반 학습 시스템 정리 |

## Collaboration Flow

GitHub 협업 흐름은 다음 순서로 진행했습니다.

1. main 브랜치에서 작업 브랜치를 생성한다.
2. 작업 브랜치에서 Markdown 파일을 작성한다.
3. 변경 사항을 commit 한다.
4. GitHub에 push 하거나 GitHub 웹에서 직접 commit 한다.
5. Pull Request를 생성한다.
6. PR 본문에 작업 내용과 체크리스트를 작성한다.
7. self review 코멘트를 남긴다.
8. 변경 내용을 확인한 뒤 main 브랜치로 merge 한다.
