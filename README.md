# Backend Practice Archive

Java/Spring 백엔드 부트캠프 과정에서 진행한 자기소개, 실습, Git 협업 흐름, 학습 기록을 정리하는 저장소입니다.

이 저장소는 단순 과제 제출용 저장소가 아니라, 부트캠프 기간 동안 쌓은 실습과 문제 해결 과정을 백엔드 취업 포트폴리오로 확장하기 위해 만들었습니다.

## Purpose

- Java/Spring 백엔드 학습 과정을 기록합니다.
- API 명세서 작성, Postman, 개발자 도구 분석 등 실습 내용을 정리합니다.
- Git branch, Pull Request, self review 흐름을 연습합니다.
- 학습 기록을 단순 메모가 아니라 포트폴리오 자료로 발전시킵니다.

## Branch Plan

| Branch | Purpose |
| --- | --- |
| `main` | 최종 결과물이 병합되는 기본 브랜치 |
| `profile-foundation` | 기본 자기소개와 현재 학습 상태 정리 |
| `backend-motivation` | 백엔드 개발자를 목표로 하는 이유 정리 |
| `backend-practice-archive` | 부트캠프 실습과 과제 기록 아카이브 |
| `git-collaboration-flow` | Git branch, commit, Pull Request, self review 흐름 정리 |
| `learning-system` | TIL, Notion, 기술 블로그 등 학습 기록 시스템 정리 |

## Current Practice

현재 진행 중인 주요 실습은 다음과 같습니다.

- API 명세서 작성
- Postman Mock Server 실습
- Chrome DevTools Network 요청 분석
- OPTIONS 요청과 GET 요청의 차이 이해
- GitHub Branch / Pull Request / Self Review 실습

## Learning Direction

백엔드 개발은 사용자의 요청이 서버로 전달되고, 데이터가 처리되어 다시 응답으로 돌아오는 흐름을 설계하는 일이라고 생각합니다.

현재는 Java와 Spring을 중심으로 HTTP, API, 데이터베이스, Git 협업 흐름을 학습하고 있으며, 각 실습을 기록으로 남겨 실제 개발 역량을 설명할 수 있는 포트폴리오로 발전시키고 있습니다.

## Review Rule

각 브랜치 작업 후 Pull Request를 만들고, 다음 기준으로 self review를 작성합니다.

- 이번 브랜치에서 무엇을 추가했는가
- 어떤 백엔드 역량과 연결되는가
- 현재 부족한 점은 무엇인가
- 다음 실습에서 어떻게 확장할 수 있는가

## Goal

부트캠프가 끝나는 2026년 11월 18일까지 이 저장소를 실습 기록, 문제 해결 과정, 협업 경험이 연결된 백엔드 취업 포트폴리오로 발전시키는 것이 목표입니다.
