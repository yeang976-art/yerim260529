# Learning System

## Overview

이 문서는 Java/Spring 백엔드 부트캠프 과정에서 학습 내용을 어떻게 기록하고 정리하는지 설명하는 문서입니다.

학습 내용을 단순히 듣고 끝내는 것이 아니라, TIL, Notion, GitHub 기록으로 연결해 다시 설명할 수 있는 형태로 정리하는 것이 목표입니다.

## Learning Flow

현재 학습 흐름은 다음과 같습니다.

1. 수업에서 개념을 학습한다.
2. 실습을 진행한다.
3. 막힌 부분과 오개념을 기록한다.
4. TIL로 하루 학습 내용을 정리한다.
5. 필요한 내용은 GitHub 문서나 기술 기록으로 확장한다.
6. 이후 프로젝트 경험과 연결해 포트폴리오 자료로 발전시킨다.

## Tools

| Tool | Role |
| --- | --- |
| Notion | 수업 내용, TIL, 과제 진행 상황 정리 |
| GitHub | 실습 기록, Markdown 문서, PR 기반 협업 흐름 기록 |
| Tistory | Java/Spring 개념을 기술 블로그 형태로 정리 |
| Postman | API 요청/응답 실습과 Mock Server 테스트 |
| Chrome DevTools | Network 요청 분석과 API 동작 확인 |
| AI Tool | 초안 작성, 구조화, 개념 검토 보조 |

## TIL System

TIL은 단순히 오늘 배운 내용을 나열하는 것이 아니라, 다음 기준으로 정리합니다.

- 오늘 무엇을 학습했는가
- 어떤 부분에서 막혔는가
- 어떤 방법으로 해결하려고 했는가
- 다시 이해한 내용은 무엇인가
- 다음 학습으로 무엇을 이어갈 것인가

## Practice Archive

GitHub에는 부트캠프에서 진행한 실습을 문서로 남깁니다.

현재는 API 명세서, Postman Mock Server, DevTools API 분석, Git/GitHub 협업 흐름을 중심으로 기록하고 있습니다.

앞으로 Spring Controller 구현, 데이터베이스, JPA, 팀 프로젝트 경험이 추가되면 이 기록을 더 구체적인 백엔드 포트폴리오로 확장할 예정입니다.

## AI Usage

AI 도구는 학습과 기록을 보조하는 도구로 사용합니다.

초안 작성, 문장 구조화, 개념 점검, 오개념 확인에는 도움을 받지만, 최종 주제 선정, 실습 수행, 기술 검토, 제출 판단은 직접 수행합니다.

AI를 사용하더라도 학습의 핵심은 직접 이해하고, 직접 실행하고, 자신의 언어로 다시 설명하는 것이라고 생각합니다.

## What I Learned

기록은 단순히 공부한 내용을 저장하는 일이 아니라, 내가 무엇을 이해했고 어디에서 막혔는지 확인하는 과정입니다.

특히 GitHub와 TIL을 함께 사용하면 실습 결과뿐만 아니라 문제 해결 과정까지 남길 수 있습니다.

이런 기록은 이후 자기소개서, 면접 답변, 포트폴리오 설명으로도 연결될 수 있습니다.

## Next Step

앞으로는 매일의 학습 기록을 TIL로 정리하고, 중요한 실습은 GitHub 문서나 프로젝트 코드로 확장할 예정입니다.

부트캠프가 진행되면서 팀 프로젝트와 Spring 기반 실습이 추가되면, 학습 기록을 실제 백엔드 개발 경험 중심으로 더 구체화할 계획입니다.
